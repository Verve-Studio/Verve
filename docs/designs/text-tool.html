Transparent Overlay
The contenteditable / textarea element sits on top of the rendered text with background: transparent, border: none. Only the cursor (1.5px white bar) and selection highlight (30% accent blue) are drawn on top. The rasterized text always shows through.
Bounding Box
1px dashed #0699fb border drawn on the canvas overlay. Eight 7×7px white square handles with 1px accent border at all cardinal and corner positions. Handles stay 7×7px in screen space regardless of zoom. Hidden when not in editing mode.
Overflow Indicator
13×13px square at the bottom-right of the bounding box, just outside the border. Accent blue + symbol. Visible only for area text (boxWidth > 0) when text exceeds boxHeight. Drawn on overlay, not written to layer pixels.
Point Text vs Area Text
Point text (click): boxWidth = 0, auto-size box, no wrapping, no overflow indicator. Dragging any handle converts to area text. Area text (drag): fixed boxWidth × boxHeight, word-wrap, text clipped at bottom, overflow indicator when content exceeds box.

T icon: Georgia 20px bold, 26×26px, same slot as the pixel thumbnail — no thumbnail for text layers. Layer name: auto-derived from content (first 30 chars); updates live while typing. Editing dot: small accent-blue dot at bottom-right of T icon while in editing mode. Missing font: amber ! badge — hover shows substituted font name. Locked: lock icon replaces opacity badge; layer is dimmed, Text tool cannot enter editing mode. Hidden: strikethrough eye icon, name italicized and muted. Double-clicking the row (not the name) enters editing mode.
