Design Decisions & Ambiguities
Mode toggle placement
Mode toggle is the first control in the panel body (above Frequency), separated by a row-sep divider from the Frequency row. This follows the spec's ordering and means the toggle is always immediately visible. Alternative: place it in the title bar — rejected because the title bar is shared with icon, title, and close button, leaving insufficient room for a two-button segmented control.
Channel label color coding
C/M/Y/K labels use muted CMYK color approximations against the dark surface: C=#3dbcbc, M=#bc4d8c, Y=#a0a000, K=#909090. These are toned down from pure CMYK ink colors to remain legible and professional on the dark theme. The labels are single letters (C/M/Y/K) — not "C Offset" etc. — because the "Channel Offsets" section header already establishes the context.
Frequency unit label
The spec states the unit is "cells/100px". Displayed as "c/100px" (abbreviated) to avoid overflow in the compact row layout. Full "cells/100px" is used in the status bar where space is less constrained. The abbreviation is consistent with standard compact panel usage in professional tools.
Channel row height vs. Frequency row
Frequency row uses the standard 28px height (slider-row). Channel offset rows use 24px (chan-row) to keep the CMYK panel compact — four 24px rows adds only 96px vs 112px at 28px. This keeps the panel within a reasonable floating window size without scrolling. Both row heights are consistent with the density-first PixelShop design language.
Offset slider gradient direction
The spec does not specify how ±50% sliders should be visually represented. This design uses a center-origin gradient where the accent fill grows outward from 50% (the zero point) in whichever direction the value is non-zero. At value=0 the track shows no accent fill. This is more intuitive than fill-from-left (which would show 50% blue at the neutral position). Implementation: CSS custom properties --off-lo/--off-hi updated via oninput JavaScript.
Channel offsets: hidden vs. preserved in B&W
The spec states: "per-channel offsets are hidden (but preserved) when switching to B&W." The design hides the entire cmyk-section div (display:none in B&W mode) while JS keeps the offset values in memory. When switching back to Color (CMYK) mode, the previously entered values are restored. This matches spec intent exactly.
