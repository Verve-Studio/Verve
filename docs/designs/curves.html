Design Decisions & Flags
Panel Hierarchy

Curves is intentionally wider than the existing two-slider adjustments so the graph remains the primary control and point telemetry fits without wrapping. Dense rows keep the panel desktop-native instead of turning it into a modal inspector. layout decision

Channel, preset, graph, telemetry, aids, then actions matches the expected order of use: select target, shape tone, inspect point, manage settings.

Preview vs. Visibility

Preview is a panel-local bypass and stays inside the Curves panel footer area. The layer eye remains in the Layers panel and can still suppress output even when Preview is on. The prototype exposes both states separately because they serve different mental models. important distinction

Point Editing Affordance

Selected-point telemetry uses explicit In / Out fields instead of only a transient tooltip so keyboard nudging, direct numeric entry, and delete behavior are legible in a compact panel. Endpoint delete is intentionally disabled rather than hidden to teach the rule. interaction decision

Secondary States

Histogram failure and invalid paste are treated as non-blocking: the graph stays usable, and the error surfaces as a toast rather than a modal. Linear identity gets a subtle empty-state hint inside the graph because the diagonal alone is easy to misread as inactive. edge-case handling
