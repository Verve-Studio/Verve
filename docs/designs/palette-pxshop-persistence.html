Palette Persistence in .verve Files No UI Changes

Reference document — file format change only  ·  Spec: docs/specifications/palette-verve-persistence.md
Summary. This feature adds the swatch palette to the .verve project file format. No new dialogs, panels, or controls are introduced. Saving and opening a project already works the same way from the user's perspective — the palette is now silently included in the round-trip.
File Format Versions
version field swatches field When produced Open behavior
1 legacy absent All files saved before this feature Leave swatches at application default. No error.
2 current swatches: [{r,g,b,a}, …] Every save after this feature ships Replace current swatches with the saved array.
JSON Schema Diff
Version 1 — before
{ "version": 1, "canvas": { "width": 800, "height": 600 }, "layers": [ { "id": "lyr-1", "name": "Background", "visible": true, "opacity": 1, "data": "<base64>" } ] // no swatches key }
Version 2 — after
{ "version": 2, "canvas": { "width": 800, "height": 600 }, "layers": [ { "id": "lyr-1", "name": "Background", "visible": true, "opacity": 1, "data": "<base64>" } ], "swatches": [ { "r": 255, "g": 0, "b": 0, "a": 255 }, { "r": 0, "g": 128, "b": 255, "a": 255 }, { "r": 34, "g": 197, "b": 94, "a": 128 } ] }

Highlighted lines are the only changes to the file format. The canvas and layers structures are untouched. All channel values are integers 0–255. Alpha is always included; semi-transparent swatches (a < 255) are fully preserved.

User-Visible Behavior
Action Before this feature After this feature UI feedback
File → Save / Save As… Writes canvas + layers Writes canvas + layers + swatches None (silent)
File → Open (v2 file) — Restores canvas, layers, and swatches None (silent)
File → Open (v1 file) Restores canvas + layers; swatches unchanged Same — swatches left at app default None (silent)
Save with empty palette — Writes "swatches": [] None (silent)
Open file with empty palette — Swatches panel shows empty list None (silent)
Error Handling (Version 2 Files Only)
  • swatches key missing Abort open. Current swatches remain unchanged. Error surfaced to user.
  • Channel value out of range (<0 or >255) Abort open. Current swatches remain unchanged. Error surfaced to user.
  • Channel value non-integer (e.g. 128.5) Abort open. Current swatches remain unchanged. Error surfaced to user.
  • All other corruption Handled by existing open-file error path. Application does not crash.

Version 1 files never trigger palette validation — they have no swatches key and the palette is simply left untouched.

Edge Cases & Constraints
Backward compat Version 1 files are never migrated automatically. The format version only advances when the user explicitly saves.
Semi-transparent swatches Alpha channel (0–255) is always serialized and must survive the round-trip byte-for-byte. It must not be silently set to 255.
Swatch order Persisted in logical application state order, not the hue-sorted display order. Hue grouping is applied at display time after restore.
Large palettes Serializing hundreds of swatches as JSON is expected to be instantaneous. No progress indicator required.
Independence from .palette files This feature does not affect the standalone .palette format (Palette File I/O spec). The two persistence mechanisms are fully independent.
