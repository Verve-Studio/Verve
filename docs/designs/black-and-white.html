Design decisions
Label hue colors: Each channel label is rendered in a desaturated version of its named hue. Pure saturated values (e.g. #ffff00 for Yellows) are illegible on the dark surface. Chosen values: Reds #d05050, Yellows #b8a828, Greens #50a050, Cyans #30a8a8, Blues #5088c8, Magentas #b850b8. Font-weight 500 (medium) provides additional differentiation from plain dim labels.
Zero-tick position: The zero reference tick is at 40% from the left (not 50%), because the range is −200..+300 and zero maps to (200/500)×100 = 40%. Default values are all slightly above zero (40–80), so the thumbs open to the right of the tick — visually communicating "positive weights" at rest.
Slider fill pattern: Standard accent-fill pattern (left edge → thumb) is retained for B&W sliders rather than a hue gradient. The label color already encodes the hue identity; adding a colored gradient track would create visual noise across 6 concurrent sliders.
Canvas grayscale: CSS filter: grayscale(1) is applied immediately on panel open (adjustment active). Toggling the eye icon removes the filter, revealing the original color canvas. The brightness component of the interactive JS preview shifts the overall brightness proportionally to the global average of slider values — a prototype-only approximation.
Reset behavior: Reset restores all six sliders to their spec defaults (40, 60, 40, 60, 20, 80) rather than zeroing them. This matches Photoshop's B&W reset convention and ensures the output returns to a neutral luminosity mapping.
Panel width: 244px (vs 236px for Brightness/Contrast) to give "Magentas" (longest label, ≈52px at 11px font-weight:500) comfortable room, leaving ~118px of slider track.
