Both Tiled Mode and Show Tile Grid have checkmarks. The grid checkmark uses the tile-seam cyan color as a visual cue distinguishing it from the mode checkmark.

Design decisions
  • Tiles share zero gap between them — seam-checking is the point, so visual tile boundaries must come solely from the grid overlay, not a structural gap. A gap would introduce false seam appearance.
  • Grid overlay color is #00dcc8 (cyan) at ~55% opacity with a 0.75 px stroke. Cyan is maximally distinct from the warm browns/greens of typical texture content, and the low opacity ensures it does not mask pixel detail at the seam.
  • "Show Tile Grid" is disabled in Normal Mode to make it unambiguous that the option only applies to the tiled view. When disabled it renders at 35% opacity and the cursor is not-allowed.
  • A small "TILED" badge appears in the top bar center next to the document title whenever Tiled Mode is active. This gives a persistent, non-intrusive mode indicator without modifying any toolbar or status bar layout.
  • Fit to Window (Ctrl+0) description in the spec states it should fit all 9 tiles — the status bar would show a zoom level roughly 1/3 of the Normal Mode fit-to-window zoom.
  • No visual distinction between the center tile and the 8 surrounding tiles by default (per spec). The user identifies the editable tile by position. The grid overlay, when on, makes tile boundaries clear but does not highlight the center cell.
  • Wrap-around-capable tools (Brush, Eraser, Pencil, Clone Stamp, selection tools) are noted in the spec — no UI change is needed for these tools; they just behave differently when Tiled Mode is active.
