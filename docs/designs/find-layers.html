3 · Keyboard interactions
Shortcuts & behaviors
Alt+Shift+Ctrl+F
Win / Linux
⌥+⇧+⌘+F
macOS
Open & focus the filter bar. If already open, re-focuses without clearing the current query.
Esc
Clear text, collapse the filter bar, and restore the full layer list. Only fires while the input is focused.
Backspace ×n
Deleting all characters restores all layers while keeping the input open and focused.
✕ button
Clicking the × clear button inside the input removes the text and restores all layers; input stays open.
Any key
while input is focused
Typing goes into the filter. Tool shortcuts are suppressed — pressing B does not switch to Brush.
Click outside
Blurs the input but does not clear or collapse the filter. The last query remains active.
Tab switch
Switching document tabs resets and collapses the filter for both tabs. Filter does not persist across sessions.
Pure display filter
The filter never changes layer order, visibility (visible property), pixel data, or any other layer attribute. Hidden layers are only hidden in the panel UI — they remain fully intact in the document.
Active layer during filter
If the active layer does not match the current query, it is hidden in the panel but remains active for all tool operations. It is never automatically changed.
Group surfacing rules
A group row stays visible if any descendant matches — even if the group name itself does not. If the group name matches, all its descendants are shown regardless of their individual names.
